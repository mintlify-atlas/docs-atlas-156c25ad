## Overview

The User Management API provides endpoints for creating, reading, updating, and deleting user accounts. All endpoints require JWT authentication and return user data without sensitive password information.

<Note>
All endpoints in this section require a valid JWT token passed in the `Authorization` header as `Bearer {token}`.
</Note>

---

## Get All Users

Retrieve a list of all users in the system.

<CodeGroup>
```bash cURL
curl -X GET https://localhost:8443/v1/api/users \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://localhost:8443/v1/api/users', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

const users = await response.json();
```
</CodeGroup>

### Response

<ResponseField name="users" type="array">
  Array of UserDTO objects
  
  <Expandable title="UserDTO properties">
    <ResponseField name="dni" type="string" required>
      User's national identification number (DNI)
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Full name of the user
    </ResponseField>
    
    <ResponseField name="number" type="integer">
      User's phone number (9 digits)
    </ResponseField>
    
    <ResponseField name="email" type="string" required>
      User's email address
    </ResponseField>
    
    <ResponseField name="admin" type="boolean">
      Whether the user has administrator privileges
    </ResponseField>
    
    <ResponseField name="roles" type="array">
      List of role strings assigned to the user (e.g., ["ROLE_USER", "ROLE_ADMIN"])
    </ResponseField>
  </Expandable>
</ResponseField>

<Accordion title="Example Response">
```json
[
  {
    "dni": "11223344C",
    "name": "John Doe",
    "number": 612345678,
    "email": "john.doe@example.com",
    "admin": false,
    "roles": ["ROLE_USER"]
  },
  {
    "dni": "99887766B",
    "name": "Jane Smith",
    "number": 698765432,
    "email": "jane.smith@example.com",
    "admin": true,
    "roles": ["ROLE_USER", "ROLE_ADMIN"]
  }
]
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 200 | Successfully retrieved list of users |
| 403 | Forbidden - Access denied (invalid or missing JWT token) |

## Get User by ID

Retrieve a specific user by their DNI (national identification number).

<ParamField path="id" type="string" required>
  DNI of the user to retrieve (e.g., "11223344C")
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET https://localhost:8443/v1/api/users/11223344C \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const dni = '11223344C';
const response = await fetch(`https://localhost:8443/v1/api/users/${dni}`, {
  method: 'GET',

    'Authorization': 'Bearer YOUR_JWT_TOKEN'

});

const user = await response.json();
```
</CodeGroup>

<ResponseField name="user" type="UserDTO">

    <ResponseField name="name" type="string">
      Full name of the user
    </ResponseField>
    
    <ResponseField name="number" type="integer">
      User's phone number (9 digits)
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="admin" type="boolean">
      Administrator status
    </ResponseField>
    
    <ResponseField name="roles" type="array">
      List of assigned roles
    </ResponseField>
  </Expandable>
</ResponseField>

<Accordion title="Example Response">
```json
{
  "dni": "11223344C",
  "name": "John Doe",
  "number": 612345678,
  "email": "john.doe@example.com",
  "admin": false,
```

## Create User

Create a new user account with the provided details.

<CodeGroup>
```bash cURL
curl -X POST https://localhost:8443/v1/api/users \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "dni": "11223344C",
    "name": "John Doe",
    "number": 612345678,
    "password": "securePassword123",
    "email": "john.doe@example.com",
    "admin": false,
    "roles": ["ROLE_USER"]
  }'
```

```javascript JavaScript
const userData = {
  dni: '11223344C',
  name: 'John Doe',
  number: 612345678,
  password: 'securePassword123',
  email: 'john.doe@example.com',
  admin: false,
  roles: ['ROLE_USER']
};

const response = await fetch('https://localhost:8443/v1/api/users', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(userData)
});

const newUser = await response.json();
```
</CodeGroup>

### Request Body

<ParamField body="dni" type="string" required>
  User's national identification number (unique identifier)
</ParamField>

<ParamField body="name" type="string" required>
  Full name of the user
</ParamField>

<ParamField body="number" type="integer">
  User's phone number (must be 9 digits)
</ParamField>

<ParamField body="password" type="string" required>
  User's password (will be encrypted before storage)
</ParamField>

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="admin" type="boolean">
  Whether the user should have administrator privileges (default: false)
</ParamField>

<ParamField body="roles" type="array">
  List of role strings to assign to the user (e.g., ["ROLE_USER"])
</ParamField>

### Response

Returns the created user as a UserDTO object (password is not included in response).

<Accordion title="Example Response">
```json
{
  "dni": "11223344C",
  "name": "John Doe",
  "number": 612345678,
  "email": "john.doe@example.com",
  "admin": false,
  "roles": ["ROLE_USER"]
}
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 201 | User created successfully |
| 400 | Invalid input (e.g., duplicate DNI, invalid email) |
| 403 | Forbidden - Access denied |

## Update User

Update an existing user's information.

<ParamField path="id" type="string" required>
  DNI of the user to update (e.g., "11223344C")
</ParamField>

<CodeGroup>
```bash cURL
curl -X PUT https://localhost:8443/v1/api/users/11223344C \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "dni": "11223344C",
    "name": "John Updated",
    "number": 612999888,
    "password": "newPassword123",
    "email": "john.updated@example.com",
    "admin": false,
    "roles": ["ROLE_USER"]
  }'
```

```javascript JavaScript
const dni = '11223344C';
const updateData = {
  dni: '11223344C',
  name: 'John Updated',
  number: 612999888,
  password: 'newPassword123',
  email: 'john.updated@example.com',
  admin: false,
  roles: ['ROLE_USER']
};

const response = await fetch(`https://localhost:8443/v1/api/users/${dni}`, {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(updateData)
});

const updatedUser = await response.json();
```
</CodeGroup>

### Request Body

<ParamField body="dni" type="string" required>
  User's DNI (should match the path parameter)
</ParamField>

<ParamField body="name" type="string" required>
  Updated full name
</ParamField>

<ParamField body="number" type="integer">
  Updated phone number
</ParamField>

<ParamField body="password" type="string" required>
  Updated password
</ParamField>

<ParamField body="email" type="string" required>
  Updated email address
</ParamField>

<ParamField body="admin" type="boolean">
  Updated administrator status
</ParamField>

<ParamField body="roles" type="array">
  Updated list of roles
</ParamField>

### Response

Returns the updated user as a UserDTO object.

<Accordion title="Example Response">
```json
{
  "dni": "11223344C",
  "name": "John Updated",
  "number": 612999888,
  "email": "john.updated@example.com",
  "admin": false,
  "roles": ["ROLE_USER"]
}
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 200 | User updated successfully |
| 404 | User not found |
| 400 | Invalid input |
| 403 | Forbidden - Access denied |

## Delete User

Delete a user account by DNI.

<ParamField path="id" type="string" required>
  DNI of the user to delete (e.g., "11223344C")
</ParamField>

<CodeGroup>
```bash cURL
curl -X DELETE https://localhost:8443/v1/api/users/11223344C \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const dni = '11223344C';
const response = await fetch(`https://localhost:8443/v1/api/users/${dni}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

if (response.status === 204) {
  console.log('User deleted successfully');
}
```
</CodeGroup>

### Response

No content is returned on successful deletion.

### Status Codes

| Code | Description |
|------|-------------|
| 204 | User deleted successfully |
| 404 | User not found |
| 403 | Forbidden - Access denied |

## Authentication

All User Management endpoints require JWT authentication. Include your token in the Authorization header:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

To obtain a JWT token, use the [Authentication API](/api/authentication) login endpoint.

## Error Responses

All endpoints may return the following error responses:

<ResponseField name="error" type="string">
  Error type (e.g., "Forbidden", "Not Found", "Bad Request")
</ResponseField>

<ResponseField name="message" type="string">
  Detailed error message
</ResponseField>

<Accordion title="Example Error Response">
```json
{
  "error": "Forbidden",
  "message": "Access denied - Invalid JWT token"
}
```
</Accordion>

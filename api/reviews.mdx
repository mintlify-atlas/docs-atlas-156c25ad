## Overview

The Review Management API provides endpoints for creating, reading, updating, and deleting reviews. Reviews allow users to rate and comment on their experiences at housing listings.

<Note>
All endpoints require JWT authentication via the `Authorization: Bearer {token}` header.
</Note>

---

## Get All Reviews

Retrieve a list of all reviews in the system.

<CodeGroup>
```bash cURL
curl -X GET https://localhost:8443/v1/api/reviews \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://localhost:8443/v1/api/reviews', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

const reviews = await response.json();
```
</CodeGroup>

### Response

<ResponseField name="reviews" type="array">
  Array of ReviewDTO objects
  
  <Expandable title="ReviewDTO properties">
    <ResponseField name="reviewId" type="integer" required>
      Auto-generated unique identifier for the review (read-only)
    </ResponseField>
    
    <ResponseField name="rating" type="integer" required>
      Numeric rating given by the user (typically 1-5)
    </ResponseField>
    
    <ResponseField name="comment" type="string" required>
      Text comment/review written by the user
    </ResponseField>
    
    <ResponseField name="hotelCode" type="integer" required>
      Code/ID of the housing/hotel being reviewed
    </ResponseField>
    
    <ResponseField name="userName" type="string" required>
      Name of the user who wrote the review
    </ResponseField>
    
    <ResponseField name="userDni" type="string" required>
      DNI (national ID) of the user who wrote the review
    </ResponseField>
  </Expandable>
</ResponseField>

<Accordion title="Example Response">
```json
[
  {
    "reviewId": 1,
    "rating": 5,
    "comment": "Excellent hotel with amazing views! The staff was very friendly and helpful.",
    "hotelCode": 1,
    "userName": "John Doe",
    "userDni": "11223344C"
  },
  {
    "reviewId": 2,
    "rating": 4,
    "comment": "Great location in the city center. Clean and comfortable rooms.",
    "hotelCode": 2,
    "userName": "Jane Smith",
    "userDni": "99887766B"
  }
]
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 200 | Successfully retrieved list of reviews |
| 403 | Forbidden - Access denied |

</ParamField>

<CodeGroup>

```javascript JavaScript
const reviewId = 1;
```

| Code | Description |
|------|-------------|

## Create Review

Create a new review with the provided details.

<CodeGroup>
```bash cURL
curl -X POST https://localhost:8443/v1/api/reviews \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 5,
    "comment": "Wonderful experience! Highly recommended.",
    "hotelCode": 1,
    "userName": "John Doe",
    "userDni": "11223344C"
  }'
```

```javascript JavaScript
const reviewData = {
  rating: 5,
  comment: 'Wonderful experience! Highly recommended.',
  hotelCode: 1,
  userName: 'John Doe',
  userDni: '11223344C'
};

const response = await fetch('https://localhost:8443/v1/api/reviews', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(reviewData)
});

const newReview = await response.json();
```
</CodeGroup>

### Request Body

<ParamField body="rating" type="integer" required>
  Numeric rating (typically 1-5, where 5 is the highest rating)
</ParamField>

<ParamField body="comment" type="string" required>
  Text review/comment describing the user's experience
</ParamField>

<ParamField body="hotelCode" type="integer" required>
  Code/ID of the housing/hotel being reviewed (must be an existing housing listing)
</ParamField>

<ParamField body="userName" type="string" required>
  Name of the user writing the review
</ParamField>

<ParamField body="userDni" type="string" required>
  DNI of the user writing the review (must be an existing user)
</ParamField>

<Info>
The `reviewId` field is auto-generated and should not be included in the request body.
</Info>

### Response

Returns the created review with an auto-generated reviewId.

<Accordion title="Example Response">
```json
{
  "reviewId": 3,
  "rating": 5,
  "comment": "Wonderful experience! Highly recommended.",
  "hotelCode": 1,
  "userName": "John Doe",
  "userDni": "11223344C"
}
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 201 | Review created successfully |
| 400 | Invalid input (e.g., invalid rating, non-existent hotel or user) |
| 403 | Forbidden - Access denied |

## Update Review

Update an existing review.

<ParamField path="id" type="integer" required>
  ID of the review to update (e.g., 1)
</ParamField>

<CodeGroup>
```bash cURL
curl -X PUT https://localhost:8443/v1/api/reviews/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 4,
    "comment": "Updated review: Still a great place, but the breakfast could be improved.",
    "hotelCode": 1,
    "userName": "John Doe",
    "userDni": "11223344C"
  }'
```

```javascript JavaScript
const reviewId = 1;
const updateData = {
  rating: 4,
  comment: 'Updated review: Still a great place, but the breakfast could be improved.',
  hotelCode: 1,
  userName: 'John Doe',
  userDni: '11223344C'
};

const response = await fetch(`https://localhost:8443/v1/api/reviews/${reviewId}`, {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(updateData)
});

const updatedReview = await response.json();
```
</CodeGroup>

### Request Body

All ReviewDTO fields can be updated (see Create Review for field descriptions).

<Warning>
Users should only be able to update their own reviews. Ensure proper authorization checks are in place.
</Warning>

### Response

Returns the updated review.

<Accordion title="Example Response">
```json
{
  "reviewId": 1,
  "rating": 4,
  "comment": "Updated review: Still a great place, but the breakfast could be improved.",
  "hotelCode": 1,
  "userName": "John Doe",
  "userDni": "11223344C"
}
```
</Accordion>

### Status Codes

| Code | Description |
|------|-------------|
| 200 | Review updated successfully |
| 404 | Review not found |
| 400 | Invalid input |
| 403 | Forbidden - Access denied |

## Delete Review

Delete a review by its ID.

<ParamField path="id" type="integer" required>
  ID of the review to delete (e.g., 1)
</ParamField>

<CodeGroup>
```bash cURL
curl -X DELETE https://localhost:8443/v1/api/reviews/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const reviewId = 1;
const response = await fetch(`https://localhost:8443/v1/api/reviews/${reviewId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

if (response.status === 204) {
  console.log('Review deleted successfully');
}
```
</CodeGroup>

### Response

No content is returned on successful deletion.

<Warning>
Users should only be able to delete their own reviews. Administrators may delete any review.
</Warning>

### Status Codes

| Code | Description |
|------|-------------|
| 204 | Review deleted successfully |
| 404 | Review not found |
| 403 | Forbidden - Access denied |

## Get Comments for a House (Paginated)

Retrieve paginated comments/reviews for a specific house.

<Info>
This endpoint is part of the Custom AJAX API and provides pagination support for displaying reviews on the frontend.
</Info>

<ParamField path="id" type="integer" required>
  ID of the house
</ParamField>

<ParamField query="page" type="integer">
  Page number (0-based, default: 0)
</ParamField>

<ParamField query="size" type="integer">
  Number of items per page (default: 3)
</ParamField>

<CodeGroup>
```bash cURL
curl -X GET "https://localhost:8443/v1/api/rooms/1/comments/extra?page=0&size=6" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const houseId = 1;
const page = 0;
const size = 6;

const response = await fetch(
  `https://localhost:8443/v1/api/rooms/${houseId}/comments/extra?page=${page}&size=${size}`,
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_JWT_TOKEN'
    }
  }
);

const paginatedReviews = await response.json();
```
</CodeGroup>

### Response

Returns a paginated list of Review entities (full objects including hotel and user details).

<Accordion title="Example Response">
```json
{
  "content": [
    {
      "reviewId": 1,
      "rating": 5,
      "comment": "Excellent hotel with amazing views!",
      "hotel": {

        "location": "Barcelona, Spain"

        "dni": "11223344C",
        "name": "John Doe",

      }
    }
  ],
  "pageable": {
    "pageNumber": 0,
    "pageSize": 6,
    "offset": 0
  },
  "totalPages": 2,
  "totalElements": 10,
  "last": false,
  "first": true
```

### Status Codes

| Code | Description |
|------|-------------|
| 200 | Successfully retrieved paginated comments |

## Best Practices

<Tip>
**For Users:**
- Leave reviews only after completing your stay
- Be honest and constructive in your feedback
- Include specific details about your experience
- Rate fairly based on the housing category and price point

**For Developers:**
- Validate that users can only review properties they've reserved
- Implement rate limiting to prevent review spam
- Consider requiring reservations to be marked as `valorated: true` before allowing reviews
- Sanitize user input in comments to prevent XSS attacks
</Tip>
